# 00 · 页面原型决策树（新建页面第 0 步）

> 新建任何页面前，**先定平台（桌面 / 移动），再判断页面原型**，最后 copy 对应壳层、只填业务内容。
> 页面原型集合**固定为桌面 3 种（A/B/C）+ 移动 3 种（A-m/B-m/C-m）**，**不得自创第 7 种**。
> 列表 / 详情 / 状态 / edge-case 页**不是独立原型**——它们是「对话流里 AI 返回的卡片」或「抽屉 / 底部弹层里的内容」。

## 第 -1 步：先定平台（禁止临场猜）

平台按 SKILL.md「平台判定」三级来源取：**① `v5-profile.md` 平台映射 → ② 页面规划文档「平台」字段 → ③ 问用户一次并回填档案**。
禁止：用桌面壳层画移动页（80px 侧导航塞进 375 视口）、给移动页临场自创壳层、凭「感觉像 App」自行改判。

## 判断决策树（两个平台同一棵树，只是落到不同壳层）

```
这个页面……
├─ 是产品/Agent 的总入口、欢迎或概览页？        → 桌面 A 首页/门户   ｜ 移动 A-m 门户(TabBar)
├─ 只在对话中完成（含 AI 返回卡片/列表/详情卡）？ → 桌面 B 对话流      ｜ 移动 B-m 对话流
└─ 需要在对话旁持续展开详情/表单/编辑面板？      → 桌面 C 右侧抽屉    ｜ 移动 C-m 底部弹层
```

判断口诀：**先看是不是门户入口（A）；不是就默认对话流（B）；对话旁要常驻面板才升级成抽屉/弹层（C）。**
**移动端没有右侧抽屉**——桌面「对话旁展开」的一切场景，移动端一律用底部弹层（C-m）承载。

## A · 首页/门户

- **壳层**：`assets/shell/home-portal.html`
- **基准图**：`assets/figma-reference/shell-home-portal.png`（Figma 节点 `536:522`）
- **固定结构**：左导航(80) + 主区[居中内容列 `--v5-portal-col` 920]，**无顶栏**。内容列自上而下：欢迎标题 → 概览副文 → 统一首页群像 → 中央提问台 → 快捷动作卡组。群像固定使用 `assets/media/home-agent-cast.png`，完整展示并保持宽高比，不按项目 Agent 数量改图。
- **固定槽位**：
  - `SLOT: 左侧导航栏`（与对话流逐字一致，勿各写一套）+ `导航项列表` + `当前用户头像`
  - `SLOT: 欢迎/概览`
  - `SLOT: 统一首页群像`（固定 `home-agent-cast.png`；禁止替换、拆分、裁成独立头像或重新拼图）
  - `SLOT: Agent 入口（中央提问台）` + `@专员快捷 chips` + `右侧操作`；每个 chip 直接链接对应 B / C 对话首态，禁止链接功能列表或选择页
  - `SLOT: 快捷动作（入口卡组）`；标题完整显示业务名称，统一字号，不单独放大首字 / emoji / 装饰符
- **何时用**：产品首页、Agent 总入口、概览落地页。

## B · 对话流（仅对话）

- **壳层**：`assets/shell/chat-flow.html`
- **基准图**：`assets/figma-reference/shell-chat-flow.png`（Figma 节点 `1144:3888`）
- **固定结构**：两栏 = 左导航(80) ｜ 主列[顶栏(60) + 居中对话列(`--v5-content-max` 840) + 底部输入台(840×136)]。
  - **经 Figma 实证：无常驻会话列表**（旧 HomeworkChatShell 的 300px 会话列表未被采用，勿硬加）。
  - 用户气泡右对齐（浅灰底），助手消息左对齐（头像+名称+时间+正文）。
- **固定槽位**：
  - `SLOT: 左侧导航栏` + `导航项列表` + `当前用户头像`
  - `SLOT: 顶栏`（标题 + 右侧会话操作）
  - `SLOT: 对话内容区`（业务消息/结果卡/列表项——用 `components/chat-result-card.html`，不是 `components/card.html`）
  - `SLOT: 底部输入台`（用 `components/input-composer.html`）
- **何时用**：与 Agent 对话、AI 返回卡片的所有场景（含 AI 返回的列表/详情卡）。

## C · 抽屉（右侧展开）

- **壳层**：`assets/shell/drawer.html`
- **基准图**：`assets/figma-reference/shell-drawer.png`（Figma 节点 `2907:2110`）
- **固定结构**：在对话流(B)基础上，右侧展开抽屉面板，对话列随之收窄。抽屉三段式：头部(标题+关闭) / 内容(详情/表单) / 底部操作。扁平白底 + 左边框分隔，无投影。
- **两种宽度变体**（用修饰类切换）：
  - `.v5-drawer--third`：**1/3 屏抽屉** `--v5-drawer-width-third` = 500px（Figma 实测）。轻量详情用。
  - `.v5-drawer--half`：**半屏抽屉** `--v5-drawer-width-half` = 720px（Figma 无此态，规格补充 = 1440/2）。重内容/复杂表单用。
- **固定槽位**：基座同 B（导航/顶栏/对话/输入台）+ `SLOT: 抽屉标题`(含关闭) + `SLOT: 抽屉内容（详情/表单）` + `SLOT: 抽屉底部操作`。
- **何时用**：需要在对话旁看详情或填表单时。

- **壳层**：`assets/shell-mobile/portal.html`
- **结构基准**：Figma「大平台设计-UI」教师端 app 首页 `56301:15219`（file `ljg4eVP3elzzx5b2Q8FYdI`）；渲染基准 `assets/figma-reference/shell-mobile-portal.png`
- **固定结构**：375×812 画板 = 模拟状态栏(44) → 顶栏(44：logo+产品名+右操作) → 滚动内容[统一首页群像 → 快捷入口行 → 区块标题 → 入口卡组] → 底部 TabBar(49) + 安全区(34)。群像使用 `home-agent-cast.png` 等比缩放。**无左侧导航**——一级切换全靠 TabBar（3–5 项，激活项近黑加粗，不用品牌色平涂）。
- **固定槽位**：`SLOT: 顶栏` · `SLOT: 快捷入口行` · `SLOT: 区块标题` · `SLOT: 入口卡组` · `SLOT: 底部 TabBar`
- **何时用**：移动产品首页、Agent 总入口/总览页（如智能体列表）、TabBar 一级页。

## B-m · 移动对话流（H5 375）

- **壳层**：`assets/shell-mobile/chat-flow.html`
- **结构基准**：Figma 同文件对话页 `56301:10135`；渲染基准 `assets/figma-reference/shell-mobile-chat-flow.png`
- **固定结构**：模拟状态栏(44) → 顶栏(44：返回+居中标题+更多) → 消息流[日期分隔 / AI 左（头像 36+名称行+浅灰气泡）/ 用户右（浅灰底描边气泡）] → 底部输入台(胶囊输入+语音+加号) + 安全区(34)。**二级页无 TabBar**。气泡最大宽 247。
- **固定槽位**：`SLOT: 顶栏` · `SLOT: 对话内容区`（结果卡用 `components/chat-result-card.html`，卡宽 `max-width:100%` 随移动列自适应）· `SLOT: 底部输入台`
- **何时用**：移动端与 Agent 对话的一切场景（含 AI 返回卡片/列表/详情卡）。

## C-m · 移动底部弹层（H5 375）＝ 桌面 C 抽屉的移动对应物

- **壳层**：`assets/shell-mobile/sheet.html`
- **结构基准**：Figma 同文件动作面板 `56301:11808` / 工具面板 `56301:12470`；渲染基准 `assets/figma-reference/shell-mobile-sheet.png`
- **固定结构**：对话流基座（遮罩下）+ 底部弹层[顶部圆角 16，三段式：标题+关闭 / 内容 / 底部操作 + 安全区]。
- **两种变体**（修饰类二选一，勿用 JS 切换——一页一态）：
  - `.v5m-sheet--half`：**半屏内容弹层**（默认，≈60% 高）——详情 / 表单 / 列表选择。
  - `.v5m-sheet--action`：**动作面板**——纵向按钮列表（危险项红字）+ 灰带分隔 + 取消。
- **固定槽位**：基座同 B-m + `SLOT: 弹层标题`(含关闭) + `SLOT: 弹层内容` + `SLOT: 弹层底部操作`
- **何时用**：移动端对话旁看详情、填表单、多选确认、长按动作菜单。
- **居中确认弹窗仍是组件不是原型**：移动端二次确认（如删除确认）用 `components/dialog.html` 的移动窄版（宽 335 = 375−2×20），不是弹层页。

## 不是独立原型（常见误判）

| 看起来像 | 实际归属 |
|---|---|
| 列表页（课程列表、作业列表…） | B/B-m 对话流里 AI 返回的**列表卡片**，或 C 抽屉 / C-m 弹层内容 |
| 详情页（作业详情、会议详情…） | B/B-m 的**详情卡片**，或 C 抽屉 / C-m 弹层内容 |
| 状态/edge-case 页（空、错误、加载…） | 对应原型内的**态**，用 `components/` 的空状态/Toast 等表达，不另立原型 |
| 后台配置/管理页 | 超出对话原型范围；如需配置，优先用 C 抽屉承载表单。**管理后台默认桌面平台**，即便用户端是移动项目 |
| 移动端「侧滑菜单 / 汉堡抽屉」 | 不存在——移动一级切换用 A-m 的 TabBar，对话旁展开用 C-m 底部弹层 |
