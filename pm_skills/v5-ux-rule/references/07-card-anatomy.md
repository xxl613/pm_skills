# 07 · 卡片设计规范（对话流卡片统一解剖）

> 真源代码：`assets/components/chat-result-card.html`（直接 copy，禁止凭本文件文字重拼类名/取值）。
> 本文件回答"什么时候用哪种卡、槽位怎么排、哪些细节不许漂"。与 `05-ai-patterns.md` 的排布规则（一卡一对象/不撑满/无内部滚动等）叠加生效。
> 注意区分：`components/card.html` 的 `.v5-card` 是**弹窗/表单场景**的卡；本规范全部是对话流里的 `.v5-rcard` 家族。

## 统一解剖（槽位顺序固定）

所有对话流卡片共用一个外壳 `.v5-rcard`（白底 surface + 1px 描边 + 圆角 lg + `max-width:600px; align-self:flex-start`），内容按以下槽位**从上到下**排列。每个槽位可省略，但出现时顺序不许调换：

```
① 类别标签 .v5-rcard__label     （可选，左上角）
② 标题 .v5-rcard__t + meta .v5-rcard__meta （可选）
③ 内容区：左色条块 .v5-block ×N ／ 选项行 .v5-opt ×N ／ 正文段落
④ 底部操作行 .v5-rcard__acts    （可选：meta 靠左 + 按钮组靠右）
```

带表头分隔线的旧式结构（`.v5-rcard__hd` + `__bd` + `__foot`）仍然有效，适合"数据卡"（key/value 字段+来源脚注）；本规范的"标签+标题"式适合"任务/方案/问题/提醒卡"。同一张卡**二选一**，不混用。

## 卡片字号阶梯（规范定值，元素级像素）

| 层级 | 类 | 字号/行高 |
|---|---|---|
| 大标题 | `.v5-rcard__t` | 18px / 26px semibold |
| 描述性文字 | `.v5-rcard__desc` | 14px / 20px secondary |
| 内容标题 | `.v5-block__n` | 16px / 24px semibold |
| 内容正文 | `.v5-block__d` | 14px / 24px secondary |

不要给这四层各自再发明字号；卡外的消息正文仍走 `--v5-font-size-body` 等令牌。

## 三种结构变体

| 变体 | 构成 | 何时用 |
|---|---|---|
| 无按钮 | 标签+标题+描述+内容块（默认左色条） | 纯展示结果，无需用户表态 |
| 有按钮 | 上者 + 底部操作行（次按钮+近黑主按钮） | 需要用户确认/调整/发布 |
| 无装饰 | 内容块用 `--plain`（无左色条），高亮/选中块用 `--active`（绿浅底+绿描边） | 块之间无类型/层级编码需要、或要突出单个选中项 |

## 四色生命周期系统（卡片外壳底色）

卡片**外壳底色只编码生命周期状态**，四色固定映射，出现时必须同时有类别标签/状态 chip 的文字表达（不靠颜色单独说话）。

**底色默认是渐变，不是纯色平涂**：`linear-gradient(135deg, {色调 14–16% + white} 0%, var(--v5-color-surface) 65%)`——左上角略深的色调向右下过渡回白，色带只在卡片上部可见，下半卡基本回到白以保证内容对比度。四色（含默认灰）统一用这一套渐变写法，只换色调值，不改渐变角度/终止点。代码见 `chat-result-card.html` 的 `.v5-rcard`/`--pending`/`--success`/`--alert`。

| 底色 | 类 | 对应状态 | 右上状态 chip |
|---|---|---|---|
| 灰/白（默认） | `.v5-rcard` | 纯列表卡、选择类型卡、等待创建的预填卡、**结束卡** | 结束卡配 `.v5-status--done`（灰） |
| 黄 | `.v5-rcard--pending` | 创建后**待开始**（如会议）、创建后**等待确认**（如专家协作产物） | `.v5-status--todo`（黄） |
| 绿 | `.v5-rcard--success` | **创建成功**、**已发布** | `.v5-status--ok`（绿） |
| 红 | `.v5-rcard--alert` | **警告提醒**类 | 通常无 chip，靠 `--alert` 类别标签 |

- **颜色=状态，不是情绪**：不允许"这段内容比较严重所以整卡换红"——只有提醒/预警**类型**的卡才用红；其余警示信息仍走卡内文字+图标。此条**取代**早前"结果卡一律不换语义色"的旧规则：外壳变色被允许，但**只能**按这张四色表取值。
- 同一状态在同一项目里必须用同一色：不许某页"待确认"用黄、另一页用蓝。
- 卡底变色时，卡内内容块保持白底（surface），保证正文对比度。

## ① 类别标签（卡片来源/类型）

## ① 类别标签（卡片来源/类型）

- 用途：让用户一眼知道这张卡是什么——`AI生成` / `专家协作` / `问题` / `提醒` / `自动任务`…（按产品真实概念，不发明装饰性标签）。
- 默认灰底（`--v5-color-fill` + tertiary 字）；**语义变体只给真正有语义的类型**：
  - `--expert`（橙，warning 系）：多 Agent/专家协作类；
  - `--alert`（红，error 系）：提醒/预警类；
  - `--ok`（绿，success 系）：创建成功/发布类。
- 右上角可放辅助信息（头像组、状态 chip），与标签同一行用 `.v5-rcard__toprow` 包裹。
- 标签色系与卡壳底色同向（黄卡配橙标/黄 chip、绿卡配绿标、红卡配红标），不交叉混色。

## ③ 内容区三种形态

| 形态 | 类 | 何时用 | 细节 |
|---|---|---|---|
| 左色条块 | `.v5-block` | 同卡内多个**同构对象/环节/层级**（分层方案、任务步骤、简报栏目） | 左 3px 色条编码类型/层级（brand/success/warning/info 变体），**必须配文字含义，不做纯装饰**；块内=标题+右侧 meta（人数/时长）+一行说明 |
| 选项行 | `.v5-opt` | 问题卡（AI 追问用户做选择） | 全宽白底描边；**选中态 = 品牌蓝描边+品牌蓝字**（`--sel`）；自由输入项用 placeholder 样式（`--input`），不是真输入框也要长得像 |
| 正文段落 | 普通文本 | 提醒卡/说明类，无同构结构时 | 关键人/关键数字加粗，一段说完，不硬拆块 |

## ④ 底部操作行

- `.v5-rcard__acts`：meta（如"预计耗时：70s"）靠左自动推开，按钮组靠右；纯按钮时加 `--end` 右对齐。
- **卡内主操作 = 近黑胶囊** `.v5-pill--primary`（`--v5-color-text` 底白字）——与弹窗主按钮同规则；**品牌蓝留给页面级 CTA**，不进卡内操作行。
- 次操作 = 白底描边胶囊 `.v5-pill--secondary`。次在左、主在右、主按钮**只有一个**。
- 按钮文案用动词短语（"应用到课件""确认开始"），不用泛化"确定/提交"。
- 问题卡固定给「跳过 + 完成」，跳过是次、完成是主。

## 状态 chip 与日期行

- `.v5-status`：`--todo` 待开始/待确认（warning 浅底）/ `--doing` 进行中（success 浅底）/ `--ok` 已发布/已完成（success 浅底）/ `--done` 已结束（**中性灰**，结束态不占语义色）。挂在行右或卡右上，不进按钮组；与四色卡壳同步取色（见上表）。
- 日期行列表项：`.v5-reco` 家族加 `.v5-reco__row`（日期块 `.v5-reco__date` + 主体 `.v5-reco__main` + 状态 chip/操作按钮）。**条数不固定时仍遵循"列表不进固定高度卡"规则**（见 05）——日期行是列表项样式，不是把整个列表塞进一张卡的许可。

## 硬约束（自检必查）

1. 同一对话流里所有卡片共用 `.v5-rcard` 外壳——不因业务不同另起一套外壳类。
2. 槽位顺序不许调换；标签永远在标题上方，操作行永远在最后。
3. **外壳底色只按四色生命周期表取值**（灰=列表/选择/预填/结束、黄=待开始/待确认、绿=成功/发布、红=警告提醒），且必须配标签/chip 文字；不按内容情绪随意换色。
4. 卡片字号走四层阶梯（18/26 · 14/20 · 16/24 · 14/24），不各自发明字号。
5. 色条/状态 chip 的颜色都必须有含义并配文字，不靠颜色单独表达（无障碍）。
6. 卡内主操作近黑、唯一；页面级 CTA 才用品牌蓝。
7. 新增卡片形态前先对照本文件五种参考形态（任务/方案卡 · 专家协作卡 · 简报卡 · 问题卡 · 提醒卡）——能套用就套用，确实是新形态才扩展，并回填到 `chat-result-card.html` + 本文件。
