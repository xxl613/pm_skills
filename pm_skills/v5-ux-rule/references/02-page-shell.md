# 02 · 母版壳层（尺寸 / 栅格 / 复用规则）

> 页面原型的壳层是**直接 copy 的稳定骨架**（桌面 `assets/shell/*.html`，移动 `assets/shell-mobile/*.html`），不要重画。
> 先按 SKILL.md「平台判定」定平台，再按 `00-page-archetypes.md` 判断用哪个壳层。本文件给壳层的固定尺寸与复用纪律。

## 桌面壳层固定尺寸（1440 基准，全部走令牌）

| 区域 | 尺寸 | 令牌 | 出现在 |
|---|---|---|---|
| 左侧导航栏 | 宽 80，满高 | `--v5-nav-width` | A / B / C（**三壳层逐字一致**） |
| 顶栏 | 高 60，左右内距 24，底 1px 边框 | `--v5-topbar-height` | B / C |
| 对话内容列 | 宽 840，居中 | `--v5-content-max` | B / C |
| 门户内容列 | 宽 920，居中 | `--v5-portal-col` | A |
| 底部输入台 | 840 × 136 | `--v5-content-max` × 字面 136 | B / C |
| 右侧抽屉 · 1/3 屏 | 宽 500（Figma 实测） | `--v5-drawer-width-third`，壳层文件 `shell/drawer.html` | C |
| 右侧抽屉 · 半屏 | 宽 720（派生 = 1440/2，Figma 无） | `--v5-drawer-width-half`，壳层文件 `shell/drawer-half.html` | C |

- **桌面画板基准宽固定为 1440**，内部坐标、字号、组件宽度和间距都按该基准绘制；实际显示比例统一为 **`显示比例 = viewportWidth / 1440`**。必须使用 `assets/project-profile/_viewport-fit.js`，由 `sync-viewport-fit.py` 注入桌面页面；**不按高度二次缩放**，也不为 1024 / 1366 / 1920 另造断点或重新排版。
- 适配引擎按当前屏幕高度补足设计画布，保证产品区域至少覆盖可视高度；长内容按缩放后的真实高度继续纵向滚动。`_pagenav.js` 走查按钮留在缩放画布外，保持 40px 屏幕尺寸、可拖动并跨页面记忆位置。
- **移动页不是桌面页的响应式收窄**——用 `shell-mobile/` 专属壳层（见下节），`v5m-` 页面不得接入 `_viewport-fit.js`。
- 抽屉打开（C）时主列宽 = 1440 − 80 − 抽屉宽；对话列随之收窄（`min-width:0`）。

## 移动壳层固定尺寸（H5 · iPhone X 基准 375×812，走 `--v5m-*` 令牌）

结构实测自 Figma「大平台设计-UI」教师端 app（file `ljg4eVP3elzzx5b2Q8FYdI`，画布 `56301:10134`）。

| 区域 | 尺寸 | 令牌 | 出现在 |
|---|---|---|---|
| 手机画板 | 375 宽 × 812 最小高，页面居中展示 | `--v5m-viewport` × `--v5m-screen-min` | A-m / B-m / C-m |
| 模拟状态栏 | 高 44（原型演示态；上线为系统/微信原生） | `--v5m-statusbar-height` | A-m / B-m / C-m |
| 顶栏 | 高 44（A-m：logo+产品名；B-m/C-m：返回+居中标题+更多） | `--v5m-topbar-height` | A-m / B-m / C-m |
| 底部 TabBar | 高 49（3–5 项，icon 20 + 10px caption） | `--v5m-tabbar-height` | 仅 A-m（一级页） |
| Home Indicator 安全区 | 高 34（内含 134×5 胶囊条） | `--v5m-safe-bottom` | A-m / B-m / C-m |
| 页面左右内距 | 16 | `--v5m-page-pad` | 全部 |
| 消息气泡最大宽 | 247（字面值，Figma 实测） | — | B-m / C-m |
| 会话/列表行 | 高 76，头像 52（字面值，Figma 实测） | — | A-m 列表态 |
| 底部弹层 | 顶圆角 16（`--v5-radius-lg`）；半屏 ≈60% 高 | 修饰类 `.v5m-sheet--half` / `--action` | C-m |

- **移动壳层类名前缀 `v5m-`**，与桌面 `v5-` 不混用；颜色/间距/圆角/字号与桌面共用 `--v5-*` 令牌（theme.css 项目覆盖同时作用于两端），仅正文行高（`--v5m-line-height-body` 24）与顶栏标题（`--v5m-font-size-title` 17）为移动收紧值。
- **移动一级页（A-m）无左侧导航**，切换靠 TabBar；**二级页（B-m/C-m）无 TabBar**，返回靠顶栏。
- **原型固定按 375 画**，不做移动端内部的响应式档位；真实 H5 由前端按百分比自适应。

### 抽屉宽度怎么选（1/3 vs 半屏）

- **默认 1/3 屏（500，`drawer.html`）**：单列详情、简单表单、执行计划、阅读型内容——窄抽屉够用就用窄的，主列留得更宽。
- **半屏（720，`drawer-half.html`）**：内容需要更大横向空间时才升半屏——**双列表单、表格/明细、并排对照**。1/3 屏放不下才用半屏，不要默认用半屏。
- 两种宽度共用同一套壳层结构，只差根元素的修饰类：`.v5-app--third`（默认）/ `.v5-app--half`。**新建抽屉页直接 copy 对应那个文件**，不要在一个文件里用 JS 切换宽度（违反「一页一态」）。

## 左侧导航（仅桌面壳层；跨壳层一致，勿各写一套）

- 结构：顶部 logo → 图标+caption 导航项（日历 / 课程 / AI教研 / **V5小管家**[激活态：图标浅灰圆角底 + 文字加粗] / AI助教 / 知识发现 / 资源库）→ 底部用户头像。
- 新建 A/B/C 任意壳层页面时，**导航整段从已有壳层复制**（`chat-flow.html` 的 `<nav class="v5-nav">…</nav>`），只改激活项，不重写样式。三壳层导航必须 diff 一致。

## 复用纪律（来自 V5 原型约定）

1. **除中间内容区外，一律包壳层**——顶栏/导航/输入台/抽屉框架都来自壳层，不重抄，避免样式漂移。
2. **一页一态**：每个原型页静态、完整呈现该场景全部信息。**禁止页面内 `useState`/Tab/条件渲染切换同一流程的多态**（导出/截图会漏态）。
3. **多态拆多页**：同一业务的「选前/选后/空/满/错误」等拆成多个独立页面，每个 URL/目录项对应一个确定态。
4. **允许页间跳转**：`<a href="?page=…">` 或目录切换（全页刷新），不同原型页之间可跳。
5. **可评审**：新页必须能独立打开、截图、在页面目录中定位。

## 槽位填充顺序（新建页面时）

1. copy 对应壳层文件 → 2. 改导航激活项 → 3. 改顶栏标题/操作 → 4. 往内容区/抽屉槽位填业务（用 `components/` 组件）→ 5. 截图自检（见 `99-review-checklist.md`）。
