# 逐页平台路由

本规则解决“同一项目同时存在 PC 与移动端页面时，Agent 如何自行判端”。平台是页面级属性，不是项目级单选项。

## 判定顺序

| 优先级 | 证据 | 处理 |
|:---|:---|:---|
| 1 | 用户明确指定当前页/页面类别的平台 | 直接执行；高于历史档案 |
| 2 | 页面规划文档的「平台」字段 | 照抄；与用户本轮指令冲突时以用户为准 |
| 3 | `v5-profile.md` 平台映射 | 按页面类别套用，记录到逐页清单 |
| 4 | 来源证据 | Figma 画板、已有路由、设备壳、渠道、宿主容器 |
| 5 | 业务承载方式 | 轻触达/现场操作偏 mobile；密集配置/批量管理偏 desktop |

只有两条强证据冲突、且不同选择会改变主结构时才询问用户。信息不足但不冲突时，做可解释的默认判断并写入档案。

## 来源证据矩阵

| 证据 | 判定 |
|:---|:---|
| 画板宽 320–480，iOS/Android 状态栏、Home Indicator | `mobile` |
| H5、手机、公众号、小程序、移动工作台、扫码进入 | `mobile` |
| 画板宽 1280–1920、左侧导航、多栏工作台 | `desktop` |
| 管理后台、批量操作、密集表格、复杂筛选/配置 | `desktop` |
| 单个需求同时给出 375 与 1440 画板 | 建两个页面映射，不做响应式替代 |
| 只有业务描述，无画板 | 使用业务承载方式判断并记录“推断”证据 |

## 逐页档案格式

在项目 `v5-profile.md` 中维护：

```md
| 页面/页面类别 | 平台 | 证据 | 壳层 |
|:---|:---|:---|:---|
| 教师数据分析首页 | desktop | 管理工作台，1440 画板 | A |
| 教师数据分析对话 | mobile | 公众号 H5，375 画板 | B-m |
```

同一个业务流程可以在两端各有页面，但需要共用业务对象、Agent 身份、状态语义和结果口径。

## 壳层映射

| 意图 | PC | 移动端 |
|:---|:---|:---|
| 入口/门户 | A | A-m |
| 对话与结果 | B | B-m |
| 辅助操作/详情 | C 右抽屉 | C-m 整页或底部弹层 |

## 禁止混用

- 移动页禁止 `v5-nav`、`v5-drawer`、桌面 840/920 固定内容列和 `_viewport-fit.js`。
- 桌面页禁止把 `v5m-safe`、Home Indicator 或移动底部弹层作为主布局。
- 业务内容可复用，结构代码不可跨端直接复制。
- 交付前运行：

```bash
python3 scripts/check-platform.py --mobile mobile/*.html --desktop desktop/*.html
```
