# 99 · 交付前自检清单

> 每次定制原型做完后逐项核对。任何一项不过，回去改。

## 平台与页面原型
- [ ] 平台（mobile/desktop）取自 `v5-profile.md` 平台映射 / 规划文档「平台」字段，不是临场猜的
- [ ] 已先判断页面原型（桌面 A/B/C · 移动 A-m/B-m/C-m），没自创新原型
- [ ] 是 **copy 对应平台壳层**改的（移动页用 `shell-mobile/`，类名 `v5m-`），不是凭描述重画
- [ ] **没有**把桌面壳层塞进 375 视口冒充移动页（移动页不得出现 `v5-nav` 左侧导航）
- [ ] 桌面页：左侧导航与其他壳层一致（没各写一套；只改了激活项）
- [ ] 桌面 1440 画布已按视口宽度等比缩放（`scale = viewportWidth / 1440`），`sync-viewport-fit.py --check` 通过；没有再按高度缩放、另造断点或单独改字号
- [ ] A / A-m 首页完整且仅使用 `assets/media/home-agent-cast.png` 作为群像，保持宽高比；没有替换、拆分、按项目 Agent 数量重新拼图；chip / 顶栏 / 消息头像仍与 `v5-profile.md` 固定形象映射一致
- [ ] 首页输入台每个 Agent / 专员 chip 都是可点击链接，且直接进入对应 B / B-m 对话页或 C / C-m 对话首态；没有跳到功能列表、案例选择、配置或介绍页
- [ ] 首页快捷入口卡标题字号和字重一致，没有把首字、首字母、emoji 或装饰符单独放大
- [ ] 移动页：一级页有 TabBar+安全区、二级页顶栏有返回无 TabBar；底部安全区(34)与 Home 胶囊条在位；对话旁展开用 C-m 底部弹层（没画右侧抽屉/侧滑菜单）

## 令牌与视觉
- [ ] 所有颜色/字号/间距/圆角/布局尺寸走 `var(--v5-*)`，无裸值（元素级像素除外且有注释）
- [ ] 品牌蓝仅用于强调/主 CTA，主体保持灰阶
- [ ] 卡片扁平（描边+浅底），阴影只给真正浮层
- [ ] 弹窗：视口居中+遮罩；次按钮在左、主在右；弹窗内确认用近黑主按钮，页面级 CTA 用品牌蓝

## 组件复用
- [ ] 弹窗/卡片/按钮/表单/输入台/上传 都用了 `components/` 既有组件，没重抄
- [ ] 语义色配了图标/文字，不靠颜色单独表达状态

## 对话/AI（若是 B/C）
- [ ] 对话顺序：请求→主Agent判断→派发→场景Agent生成→产物→完成句→后续建议
- [ ] **交接用「拍了拍」mention 条**（`@派发方 拍了拍 @接收方 说，……`），接在派发方消息末尾，不是独立居中弱 pill；接收方 mention 用项目品牌色高亮，派发方 mention 用中性灰；链式交接（场景 Agent→场景 Agent）同样适用
- [ ] 先产物后完成句；生成态没冒充结果
- [ ] 一条消息一个说话人；Agent 用头像+职能化名称区分，**不加描述性角色副标题**；消息头各段（名称/时间/状态标签）用 `|` 分隔，且**字号统一**用 `--v5-font-size-caption`（不要各段各写一个像素值）
- [ ] 没暴露内部 workflow/系统日志/推理链
- [ ] 过程状态（已接收/思考中/调用能力/生成中）挂在消息头的 `⚡ {技能/任务名} {状态词}` 标签里，没有单独起一条 pill 消息
- [ ] AI 回答：一卡一对象、长文用文档排版不堆卡、多卡用组级 CTA
- [ ] **没有一问多卡**：数一下用户提问次数是否 ≈ 结果卡数量；用户一句话触发了好几个不同环节/维度的卡，说明该拆成多轮追问
- [ ] **卡片外壳底色只按四色生命周期系统取值**（灰=列表/选择/预填/结束、黄=待开始/待确认、绿=成功/发布、红=警告提醒，见 `07-card-anatomy.md`），且配了类别标签/状态 chip 文字；没有按内容情绪临时换色的"红绿灯"混搭；同一状态全站同色
- [ ] **卡片底色是 135deg 渐变**（色调→白），不是纯色平涂；四色统一用同一套渐变角度/终止点，只换色调
- [ ] **卡片字号走四层阶梯**：大标题 18/26、描述 14/20、内容标题 16/24、内容正文 14/24，没有各自发明字号
- [ ] **结果卡没有被拉伸撑满对话列**：`.v5-rcard` 类有 `max-width:600px; align-self:flex-start`，字段少的卡不会在 key/value 之间留大片空白
- [ ] **卡片没有内部滚动条**：条数不固定的可浏览列表/表格（如"会议列表(10)"）没有被塞进固定高度、自带滚动条的卡里；要么卡内只放前几条+总数+「查看全部」跳转列表页/抽屉，要么条数少到一屏放得下
- [ ] **多属性列表项没有写成一行纯文本拼接**（`名称 · 属性1 · 属性2`），用 `.v5-reco` 结构（标题+标签行）承载；列表项容器底色用 `var(--v5-color-surface)`，不是 `var(--v5-color-bg)`（避免看起来"透明无底"）；需要强调的徽章用实心底+白字，不与弱提示徽章（如"适配度"）共用类名
- [ ] 长文排版（非卡片的分析类回答）信息密度是否足够，不是把卡片字段简单复述成一句话
- [ ] **卡片遵循统一解剖**（见 `07-card-anatomy.md`）：槽位顺序 = 类别标签→标题+meta→内容块/选项→操作行，没有调换/混入 `.v5-rcard__hd` 表头式结构；类别标签是产品真实概念（AI生成/专家协作/问题/提醒），语义变体没有滥用
- [ ] **卡内主操作是唯一的近黑胶囊**（`.v5-pill--primary`），次操作白底描边在左；卡内没有出现品牌蓝按钮（品牌蓝只给页面级 CTA）
- [ ] 左色条块/状态 chip 的颜色都有含义并配了文字（色条编码层级/环节，chip 分 待开始/进行中/已结束，结束态用中性灰），没有纯装饰配色
- [ ] **该轮最终产物之后有一次性的回答完成操作栏**（`.v5-msg-actions`：完成状态 pill+播放文字 pill+复制/举报/点赞/点踩四个纯图标），没有在每条中间过程消息上重复出现
- [ ] **有追问建议时用 `.v5-followup` 结构**：每条是完整问句（不是关键词标签）、左侧统一图标、整行可点击浅底行，2–4 条纵向堆叠；整组下方有「关闭追问建议」；没有和「拍了拍」交接条混在同一条消息里
- [ ] 文件卡含 名称/类型/规模/状态/操作；发送后用户消息仍保留精简文件卡；**文件卡与文字气泡是两个独立元素**（各自 flex-end 堆叠），没有塞进同一个气泡边框里

## 文案
- [ ] 按钮用动词短语（结果导向），不是泛化「确定/提交」当主操作
- [ ] 错误/空状态给下一步，不只报「出错了/暂无数据」
- [ ] 无营销腔、无 PRD/路由名/评审备注混入用户可见区

## 真实内容（站在用户视角）
- [ ] **逐元素自问「上线后真实用户在本页会看到/用到它吗？」**，不会的删掉或换成真实语义
- [ ] 无「查看功能演示」「立即体验」「看看怎么用」等演示/导览入口
- [ ] 无页面自指按钮（如在「学业规划助手」页放「进入学业规划助手对话」——用户已在页内）
- [ ] 没把介绍页/落地页式的营销、导览内容当功能页
- [ ] 走查用的跳转只挂在**真实可点击元素**（导航/真实卡片/入口）上，没为走查新增假的演示按钮
- [ ] **全站走查用浮动导航器**（`_pagenav.js`+`_pagenav.pages.js`，`sync-pagenav.py --check` 通过），清单按关系分组、当前页高亮；按钮可拖动且刷新后位置保留；不靠页面内演示按钮遍历

## 结构与可评审
- [ ] 一页一态：没用页面内 useState/Tab 切换同一流程多态
- [ ] 多态拆成了多个独立页面
- [ ] 页面能独立打开、截图、在页面目录中定位

## 渲染自检（截图）
- [ ] 无空白页、无重叠、无文字溢出
- [ ] 桌面页至少在 1024、1440、1920 三种宽度截图检查：画布始终贴合视口宽度、无横向滚动；走查按钮和展开面板不被缩放或裁切
- [ ] 固定输入台在位、消息对齐正确（用户右/助手左）
- [ ] **关键交互截图 vs `assets/figma-reference/*.png` 基准图比对**，无明显偏差
