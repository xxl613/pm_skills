# html-prototype-style

用于创建或修改 HTML / React 原型页面，约束框架复用、页面尺寸、卡片、弹窗、抽屉流程、状态机制和评审辅助层。

## 适用场景

- 在已有原型工程中新增或修改页面。
- 为对话流、A2UI 卡片、抽屉流程、普通管理页、状态机制页或异常汇总页补充原型。
- 在不丢失旧入口和旧功能的前提下补字段、补状态、补交互。
- 修改已评审页面时维护页面状态和网页可见变更记录。
- 原型确认后需要同步矫正 PRD 口径。

## 关注重点

- 先复用同类框架页面或壳层组件，再补业务内容。
- 同一原型工程保持统一画布尺寸和自适应规则。
- 对话流卡片主体只读，不承载卡内编辑、选择、筛选、刷新等交互。
- 抽屉流程需要覆盖右侧承载、任务卡片、中途关闭和完成回流。
- 页面可见文案必须面向最终用户，不能出现开发者备注或原型说明。
- 完成后需要通过只读浏览器检查或无头截图确认无白屏、错位、遮挡和文字溢出。
