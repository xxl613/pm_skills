# 项目工作区导航

本项目采用结构化工作区目录，便于持续沉淀需求、设计、技术、调研、验证和会议结论。

## 进入项目

建议按以下顺序阅读：

1. `AGENTS.md`：了解协作规则和目录使用方式。
2. `MEMORY.md`：了解当前稳定口径和关键文件入口。
3. `08_LLM_WIKI/00_入口/index.md`：了解 AI 编译索引、任务路由和已消化资料状态。
4. 根据任务类型进入对应目录或原始文件。

## 目录说明

| 目录 | 用途 |
|:---|:---|
| `00_项目管理/` | 项目背景、目标、阶段计划、待确认问题、里程碑、协作信息 |
| `01_产品设计/` | PRD、需求池、用户故事、功能说明、名词表、产品规则 |
| `02_智能体能力/` | Agent、Skill、MCP、工作流、工具能力、能力资产说明 |
| `03_技术架构/` | 架构图、技术方案、数据结构、接口边界、系统依赖说明 |
| `04_业务调研/` | 访谈、竞品、业务资料、原始调研材料 |
| `05_交付验证/` | Demo 验收、测试记录、问题清单、交付材料、评审反馈 |
| `06_设计文件/` | `DESIGN.md`、原型工程、Figma 资料、截图、视觉规范、页面目录、PRD 对照信息、变更记录、设计交付物 |
| `07_会议记录/` | 原始会议记录、整理后纪要、会议结论、行动项 |
| `08_LLM_WIKI/` | 项目资料被 AI 消化后的结论层、路由层和来源追溯层；不替代原始文件 |

每个主目录内有 `_README.md`（文件清单与用途说明，进入该目录先看它）；归档文件放在各目录的 `_archive/` 中并附归档台账，默认不读。文件的归档、删除和命名规则见 `AGENTS.md` 第 6 节。

## 根目录文件分工

| 文件 | 用途 |
|:---|:---|
| `AGENTS.md` | AI 协作规则，说明先读什么、怎么使用目录、日常协作边界 |
| `MEMORY.md` | 项目长期记忆，记录稳定口径、关键决策、当前交付目标 |
| `README.md` | 项目工作区导航，本文件 |
| `08_LLM_WIKI/00_入口/index.md` | AI 任务路由入口，说明不同问题和任务应优先读取哪些 Wiki 页面与原始文件 |

## 初始化后的下一步

1. 补充 `MEMORY.md` 的项目名称、核心用户、项目定位和当前重点。
2. 在 `00_项目管理/` 中补充项目背景、阶段计划和待确认问题。
3. 在 `01_产品设计/` 中创建或迁移当前主 PRD / 需求池。
4. 如果已有设计或原型资料，放入 `06_设计文件/`。
5. 将重要既有资料逐步消化到 `08_LLM_WIKI/`，先生成来源登记、文件摘要和候选结论。
