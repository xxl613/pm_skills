# html-prototype-to-figma

Defines the workflow for converting HTML, React, Vite prototypes, local preview
sites, or rendered web pages into editable Figma designs.

## Use When

- Syncing a local prototype into Figma.
- Capturing a business UI while excluding prototype-only review tools.
- Expanding hidden scroll content and interaction states into Figma frames.
- Verifying that generated Figma output matches the browser source.
- Troubleshooting Figma MCP conversion, visibility, clipping, or asset issues.

## Main Concerns

- Confirm the target Figma file before writing.
- Capture only real user-facing product UI.
- Exclude PRD panels, route catalogs, debug controls, and review helpers.
- Use explicit browser viewport sizes.
- Preserve full scroll content and meaningful interaction states.
- Compare browser screenshots and Figma screenshots before calling the result
  complete.
