# prd-writing-style

用于撰写、改写和审查教师中心相关 PRD、功能说明、需求池和研发对齐文档。

## 适用场景

- 按教师中心 PRD 风格整理需求。
- 将口语化需求改写成可研发拆解的 PRD 条目。
- 调整已有 PRD 的承载方式、字段处理、状态机制、提示流转和异常规则。
- 将原型确认后的最新口径同步回 PRD。
- 区分对话卡片、抽屉流程、普通管理页和综合型 PRD 的写作结构。

## 关注重点

- 只写产品行为、边界、输入、输出、状态和验收标准。
- 不写接口字段、后端枚举、实现变量名和研发过程。
- P0 / P1 补齐可研发拆解的信息；P2 保持占位。
- 对话流卡片保持只读，复杂编辑和多步骤任务进入抽屉流程。
- 涉及学生数据、成绩、MCP、外部服务或高影响动作时，补充权限、安全与审计边界。
